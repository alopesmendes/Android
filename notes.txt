configuration:
configuration pour stocker les dernières opérations

envoi d'une configuration:
	envoi par intent vers un service:
	Bundle, dictionnaire, structure récusrive de Map et List en Bundle
	l'historique des versions des config est conservé
	envoyer une config partiellement

	authentification de l'expéditeur:
	token d'authentification pour chaque app

stockage des configurations:
fichiers ou base sqlite

affichage des configurations:
interface graphique pour naviguer sur les configs de toutes les apps
visualiser toutes les versions des configs (mettre en valeur les différences)
l'utilisateur doit pouvoir modifier les configs (créer une nouvelle version)

récupération d'une config:
une app envoie un intent avec des infos,
lorsque le service de Confroid reçoit la requête, il y répond par un intent avec d'autres infos,

sauvegarde et restauraiton des configs:
Confroid exporte les configs dans un fichier sur le disque,
api storage access framework
restaurer des configs depuis un fichier de sauvegarde
réaliser, sauvegarder les configs sur un service web
api workmanager pour soumettre une tâche de sauvegarde régulière du fichier vers le service
l'utilisateur peut demander manuellement la restauration partielle ou totale du fichier dispo sur le server
